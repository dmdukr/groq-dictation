## Multi-Provider Architecture (v4.0.0–v4.1.1)

### Нова концепція
Замість прив'язки до одного сервісу — **3 слоти для кожного типу** з автоматичним fallback:

```
STT:           #1 → #2 → #3  (розпізнавання мовлення)
Нормалізація:  #1 → #2 → #3  (LLM нормалізація тексту)
Переклад:      #1 → #2 → #3  (переклад: DeepL або LLM)
```

### Provider Detection Flow
1. Користувач вводить **тільки API Key**
2. По префіксу (`gsk_`, `:fx`, `AIzaSy`, `sk-`, ...) → визначаємо **сервіс** і **base_url**
3. `GET /models` → підтверджуємо сервіс + отримуємо **список моделей**
4. Якщо префікс невідомий → показуємо dropdown вибору сервісу
5. Після визначення → блокуємо dropdown (readonly)

### Підтримувані провайдери

#### STT (коннектор — OpenAI-compatible `/audio/transcriptions`)
| Провайдер | Префікс ключа | Моделі |
|-----------|---------------|--------|
| Groq | `gsk_` | whisper-large-v3, whisper-large-v3-turbo |
| OpenAI | `sk-` | whisper-1, gpt-4o-transcribe |

_Заплановані: Soniox, Deepgram, Gladia (потрібні окремі коннектори)_

#### LLM Normalization (коннектор — OpenAI-compatible `/chat/completions`)
| Провайдер | Префікс | Free tier |
|-----------|---------|-----------|
| Groq | `gsk_` | 8 ч/день |
| Google AI Studio | `AIzaSy` | 1500 req/day |
| Cerebras | `csk-` | 30 req/min |
| Mistral | авто | 1M tokens/min |
| OpenRouter | `sk-or-` | free-tier models |
| GitHub Models | `ghp_`/`github_pat_` | Rate limited |

#### Translation
| Провайдер | Тип | Формат API |
|-----------|-----|-----------|
| DeepL | Dedicated | REST `/v2/translate` |
| Будь-який LLM | OpenAI-compatible | `/chat/completions` |

### Файли провайдерів

- `src/provider_manager.py` — ProviderManager (fallback chain, get_stt_client/get_llm_client)

## Architecture

### State Machine (engine.py)
```
IDLE → RECORDING → PROCESSING → TYPING → IDLE
  ↑                                 ↓
  └─────────── ERROR ←─────────────┘
```

### Pipeline (user flow)
1. Поставити курсор у будь-яке текстове поле
2. Натиснути гарячу клавішу (hold або toggle — залежить від налаштувань)
3. Продиктувати текст (можна абзацами)
4. Відпустити/натиснути клавішу для зупинки
5. Зачекати вставки тексту
6. Якщо текст з помилкою → відредагувати вручну
7. Двічі натиснути гарячу клавішу → програма порівняє правки і навчиться

### Processing Pipeline
```
Audio (WAV) → STT Provider (#1→#2→#3)
           → Raw text
           → LLM Normalization (#1→#2→#3) + User Profile context
           → Normalized text
           → Text injection (clipboard + Ctrl+V)
```

### Translation Pipeline
```
Selected text → 2× Ctrl+C
             → DeepL / LLM Provider (#1→#2→#3)
             → Overlay window (Copy / Replace / Close)
```

## Build Pipeline (unchanged)
1. `/simplify` (3 parallel review agents)
2. `ruff check src/ --select E,W,F --ignore E501,W291,F401`
3. `bandit -r src/ -ll --quiet`
4. `pyinstaller --distpath C:/tmp/groq-dist --workpath C:/tmp/groq-work`
5. Copy dist to `C:/tmp/dist/`, copy `installer.iss` + `config.yaml`
6. Build installer: `ISCC.exe C:\tmp\installer.iss`
7. Commit, tag, push, `gh release create`
